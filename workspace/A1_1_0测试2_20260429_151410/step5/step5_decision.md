# 投资决策框架

## 核心判断

**一句话判断**：这是一家以重资产运营为核心、用AI和平台叙事包装的港口无人重载物流运输服务商，当前最大问题是代运营模式的真实财务模型未验证，导致其商业本质（轻资产科技公司 vs 重资产运营公司）和估值逻辑存在根本性错位。

**公司本质**：重资产运营型公司，核心是场景+运营，而非AI平台

**决策**：`maybe`（信心度：medium）

**核心原因**：方向靠谱（港口无人驾驶刚需明确），但代运营模式的财务模型、AI壁垒虚实、客户需求驱动力三个关键缺口未验证，需deep dive后才能判断是否值得推进。


## 值得继续看

1. **港口、园区等封闭场景的无人重载物流是自动驾驶商业化最明确的场景之一，客户降本增效安全刚需成立，且公司已部署240台车，市场份额领先。**
   → 验证了场景真实性和初步商业化能力，这是继续推进的基础。

2. **代运营模式如果财务模型成立（回本周期<4年，净利率>10%），将带来持续收入、深度客户绑定和数据飞轮，具备从项目制向平台化转型的潜力。**
   → 代运营是BP中最具想象力的部分，也是估值逻辑从重资产运营转向轻资产平台的关键变量。

3. **团队背景（清华+百度+产业专家）和增材制造降本80%的宣称，如果验证为真，将构成显著的供应链壁垒和成本优势。**
   → 团队和成本优势是公司区别于竞争对手的核心竞争力，值得验证。


## 为什么不投

1. **代运营模式重资产、现金流压力大，BP未提供任何财务模型，规模化路径存疑。如果代运营单位经济模型不健康（回本周期>5年、净利率为负），公司本质是重资产运营公司，估值需大幅下调。**
   → 代运营是核心商业模式，其财务模型不透明是最大的投资风险，可能导致估值逻辑完全错误。

2. **核心传感器、芯片依赖外部供应商（英伟达、禾赛等），自研底盘和增材制造尚未大规模验证，技术壁垒可能被高估，公司更接近系统集成商而非核心技术拥有者。**
   → 如果技术壁垒不成立，公司护城河薄弱，面临来自传统重卡主机厂和自动驾驶公司的竞争压力。

3. **客户需求可能主要受政策驱动（如智慧港口建设要求），而非纯商业ROI。如果政策退坡，收入可持续性存疑。**
   → 政策驱动型需求波动大，影响公司长期增长逻辑和估值稳定性。

- [P1] **核心传感器、芯片依赖外部供应商，自研底盘和增材制造尚未大规模验证，技术壁垒可能被高估。**

## 必问问题

1. 何总，BP中提到代运营是公司核心模式，而且业绩领先。我们想了解一下，一个典型的代运营项目，比如在宁波港部署50台车，大概的投资额、年收入和回本周期是怎样的？
   目的：验证代运营模式的真实财务模型，判断其可持续性和规模化路径。

2. 何总，BP中提到TransportGPT和AI大模型，这些AI能力现在主要用在哪些具体环节？如果现在把TransportGPT关掉，只用传统算法，你们的无人重载车辆还能稳定运营吗？效率会下降多少？
   目的：验证AI/数据能力是否构成核心壁垒，还是只是研发/交付效率工具。

3. 何总，BP中提到客户主要是港口和园区，这些客户采购你们的无人驾驶方案，主要是为了降本增效，还是因为政策要求？如果政府明天取消所有港口无人化补贴，你觉得现有客户中，有多少还会继续用你们的方案？
   目的：验证客户需求是政策驱动还是真实经济ROI驱动，判断收入的抗政策风险能力。
