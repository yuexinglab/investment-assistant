## 四、关键发现

### 1. v1.1 的优势

**① 区分"现实"和"叙事"**
- `current_business` vs `narrative_business` 的区分非常有效
- 每个叙事业务都有 `evidence`（BP原文引用）和 `confidence`（置信度）
- 投资人一眼就能看出哪些是真实业务，哪些是故事

**② 问题与判断关联**
- `must_ask_questions` 每个问题都有 `related_to` 字段
- 问题直接指向 red_flags 或关键判断点
- Step4 可以直接利用这些关联生成追问

**③ 标注来源**
- red_flags 有 `source` 字段：Step1 判断 vs BP 内容
- 区分"我的直觉"和"BP告诉我的"
- 避免把 BP 叙事当成事实

**④ confidence 意识**
- 每个字段都有 confidence 标注
- 避免过度自信，把猜测当成结论

### 2. 当前版本的问题

**① "过度分析"倾向**
- 当前版本会展开讨论（如"增材制造的规模效应"）
- 这些是 Step4/Step5 该做的事

**② 问题与判断脱节**
- 当前版本的问题是"通用问题"，不是"有判断支撑的追问"
- 缺少 related_to 关联

**③ 叙事未标注**
- 当前版本把 BP 叙事当成事实判断
- 缺少 narrative_business 的结构化区分

## 五、实验结论

### ✅ 建议进入阶段二

**v1.1 在所有 5 个维度上都优于当前版本：**

1. ✅ **更像投资人第一反应**: v1.1 简洁精准，像投资人在脑中快速过项目
2. ✅ **更清晰定义公司本质**: 一句话定位 + "不是什么"划清边界
3. ✅ **更容易生成高质量问题**: 问题与判断关联，related_to 字段明确
4. ✅ **没有"过度分析"**: 只有判断，没有展开讨论
5. ✅ **没有把未验证内容写死**: narrative_business + confidence 标注

### 关键改进点

| 改进点 | 当前版本 | v1.1版本 |
|--------|----------|----------|
| 叙事 vs 现实 | 混在一起 | 明确区分 |
| 问题关联 | 无关 | 有关联字段 |
| 判断来源 | 不标注 | source 标注 |
| confidence | 无 | 每个字段都有 |
| 输出格式 | 纯文本 | JSON结构化 |
